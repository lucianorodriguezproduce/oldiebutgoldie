# ADR-003: Blindaje de Observabilidad y Testing

## Estado
Aceptado

## Contexto
A medida que el búnker crece, la detección manual de errores es insuficiente. Se necesita una red de alerta temprana y validación automatizada.

## Decisión
Integrar un sistema de defensa de dos capas:
- **Sentry (React/Vite)**: Para observabilidad proactiva. Captura de kernel panics y kernel replayed en producción.
- **Playwright**: Para pruebas de extremo a extremo (E2E). Validación del "Camino del Guerrero" (flujo de compra) antes de cada despliegue.

## Consecuencias
- **Positivo**: Reducción drástica de pantallas blancas. Los errores se detectan antes de que el usuario los reporte.
- **Negativo**: Incremento leve en el tiempo de build/despliegue debido a la ejecución de robots de prueba.
